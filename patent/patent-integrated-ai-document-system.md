# 특허 명세서

---

## [발명의 명칭]

### AI 기반 보안 검증 및 규제 준수 기능이 통합된 지능형 문서 편집 시스템 및 그 방법

**Intelligent Document Editing System and Method with Integrated AI-Based Security Verification and Regulatory Compliance**

---

## [기술분야]

본 발명은 인공지능(AI) 기반의 문서 편집 기술에 관한 것으로, 보다 구체적으로는 대규모 언어 모델(LLM)을 활용한 콘텐츠 자동 생성, 다계층 보안 방어, 할루시네이션(환각) 검증, 및 다중 규제 프레임워크 준수 평가가 단일 문서 편집 환경 내에서 유기적으로 통합된 지능형 문서 편집 시스템 및 그 운용 방법에 관한 것이다.

본 발명은 국제특허분류(IPC) G06F 40/00(디지털 데이터 처리: 자연어 데이터 처리), G06F 21/00(보안 장치), G06N 3/08(기계 학습) 분야에 속한다.

## [배경기술]

### 1. 종래 기술의 현황

최근 대규모 언어 모델(Large Language Model, 이하 "LLM")의 급속한 발전에 따라, 문서 편집 분야에서도 AI를 활용한 자동 콘텐츠 생성 기능이 도입되고 있다. 대표적으로 마이크로소프트 코파일럿(Microsoft Copilot), 구글 제미니(Google Gemini) 등이 기존 오피스 스위트에 AI 기능을 탑재하여 문서 초안 작성, 요약, 번역 등의 기능을 제공하고 있다.

그러나 종래의 AI 통합 문서 편집 시스템은 다음과 같은 기술적 한계를 가지고 있다.

#### 1.1 보안 취약성

종래 시스템은 LLM에 대한 프롬프트 인젝션(Prompt Injection), 탈옥 공격(Jailbreak Attack), 사회공학적 공격(Social Engineering) 등 AI 특유의 보안 위협에 대한 체계적인 방어 메커니즘이 부재하다. 특히, 한국어, 일본어, 중국어 등 CJK 언어권에서의 언어학적 우회 공격(자모 분리, 초성 약어, 고어 활용, 동형 문자 치환 등)에 대한 방어 기술은 전무한 실정이다.

#### 1.2 할루시네이션 문제

LLM이 생성한 콘텐츠에는 사실과 다른 정보가 포함될 수 있는 할루시네이션(Hallucination) 현상이 불가피하게 발생한다. 종래 시스템은 생성된 콘텐츠의 사실적 정확성을 원본 문서와 대조하여 실시간으로 검증하는 메커니즘이 없거나, 있더라도 단일 계층의 단순 비교에 그치고 있어 수치 오류, 문맥 왜곡, 규제 위반 표현 등을 포괄적으로 탐지하지 못한다.

#### 1.3 규제 준수 부재

2024년 EU AI Act(인공지능법) 발효, 2026년 한국 인공지능 기본법 시행, NIST AI RMF(위험관리 프레임워크) 공포 등 AI 관련 국제 규제가 급속히 강화되고 있다. 그러나 종래의 문서 편집 시스템은 AI 사용 이력의 체계적 기록, 투명성 보장, 위험 평가, 인간 감독(Human Oversight) 등 규제 요건을 충족하기 위한 기술적 수단을 내장하고 있지 않다.

#### 1.4 기능 분절 문제

종래 기술에서 보안, 팩트체크, 규제 준수는 각각 독립된 별개의 도구로 제공되어, 문서 편집 워크플로우와 유기적으로 연동되지 않는다. 이로 인해 사용자는 복수의 도구를 번갈아 사용해야 하며, 데이터 흐름의 단절로 인한 정보 손실과 업무 효율 저하가 발생한다.

## [발명이 해결하고자 하는 과제]

본 발명은 상기한 종래 기술의 문제점을 해결하기 위하여 안출된 것으로, 다음의 기술적 과제를 해결하는 것을 목적으로 한다.

**제1 과제**: 문서 편집 환경에서 LLM 기반 콘텐츠 생성 시, 한국어를 포함한 CJK 언어 특화 공격을 방어할 수 있는 다계층 보안 방어 시스템을 제공한다.

**제2 과제**: AI가 생성한 콘텐츠의 사실적 정확성을 원본 문서와 대조하여 규칙 기반, 수치 검증, 자연어 추론(NLI), LLM 재검증의 4계층으로 실시간 검증하는 할루시네이션 탐지 메커니즘을 제공한다.

**제3 과제**: EU AI Act, 한국 AI 기본법, NIST AI RMF, OWASP LLM Top 10 등 다중 규제 프레임워크에 대한 준수 여부를 AI 활동 로그 기반으로 자동 평가하고 증거 기반 리포트를 생성하는 컴플라이언스 시스템을 제공한다.

**제4 과제**: 상기 보안 방어, 할루시네이션 검증, 규제 준수 평가를 문서 편집 워크플로우 내에 유기적으로 통합하여, 단일 사용자 인터페이스에서 문서 편집-AI 생성-보안 검증-팩트체크-규제 준수의 전체 파이프라인을 운용할 수 있는 통합 시스템을 제공한다.

**제5 과제**: 서버 장애 또는 오프라인 환경에서도 핵심 보안 및 검증 기능이 유지되는 온라인-오프라인 하이브리드 아키텍처를 제공한다.

## [과제의 해결 수단]

상기 과제를 해결하기 위하여 본 발명은 다음의 기술적 수단을 제공한다.

본 발명의 일 실시예에 따른 **AI 기반 보안 검증 및 규제 준수 기능이 통합된 지능형 문서 편집 시스템**은:

**(a) 문서 파싱 및 편집 모듈** — 한글 문서 포맷(HWPX/HWP)을 포함하는 문서 파일을 파싱하여 내부 문서 객체 모델(Document Object Model)로 변환하고, 사용자의 편집 입력을 수신하여 상기 문서 객체 모델을 갱신하는 문서 파싱 및 편집 모듈;

**(b) AI 콘텐츠 생성 모듈** — 상기 문서 객체 모델의 구조를 분석하여 문서 유형을 자동 판별하고, 헤더-콘텐츠 쌍(header-content pair) 및 테이블 관계를 추출하여 문맥 인식 프롬프트(context-aware prompt)를 구성한 후, 외부 대규모 언어 모델(LLM) API를 호출하여 콘텐츠를 생성하고, 생성된 콘텐츠를 상기 문서 객체 모델의 대응 위치에 병합하는 AI 콘텐츠 생성 모듈;

**(c) 다계층 보안 방어 모듈(AEGIS)** — 상기 AI 콘텐츠 생성 모듈의 입력 및 출력에 대하여, 신뢰 경계 검증(Trust Boundary), 의도 분석(Intent Verification), 검색증강생성 가드(RA Guard), 분류기반 검색증강(ClassRAG), 회로 차단기(Circuit Breaker), 행동 분석(Behavioral Analysis)의 6개 방어 계층을 순차적으로 적용하는 다계층 보안 방어 모듈로서, 한국어 언어학적 우회 공격에 특화된 12개의 탐지 서브모듈을 포함하는 다계층 보안 방어 모듈;

**(d) 할루시네이션 검증 모듈(TruthAnchor)** — 상기 AI 콘텐츠 생성 모듈이 생성한 콘텐츠에서 개별 주장(claim)을 추출하고, 각 주장에 대하여 가드레일 규칙 검증, 수치 교차검증, 자연어 추론(NLI) 모델 기반 의미 검증, LLM 재검증의 4계층 파이프라인을 적용하여 지지(supported)/반박(contradicted)/중립(neutral)의 판정 및 다차원 신뢰도 점수를 산출하는 할루시네이션 검증 모듈;

**(e) 규제 준수 평가 모듈** — 상기 (b) 내지 (d) 모듈의 동작 이력을 활동 로그(Activity Log)로 기록하고, 복수의 규제 프레임워크(EU AI Act, 한국 AI 기본법, NIST AI RMF, OWASP LLM Top 10)에 대한 선언적 평가 규칙을 적용하여 준수 여부를 자동 판정하며, 각 평가 항목에 대한 증거(evidence) 및 교정 지침(remediation)을 포함하는 컴플라이언스 리포트를 생성하는 규제 준수 평가 모듈;

**(f) 통합 제어 모듈** — 상기 (a) 내지 (e) 모듈 간의 데이터 흐름을 오케스트레이션하여, 사용자의 AI 콘텐츠 생성 요청 시 보안 검증 → AI 생성 → 출력 검증 → 할루시네이션 검증 → 컴플라이언스 로깅의 파이프라인을 자동 실행하는 통합 제어 모듈;

을 포함하는 것을 특징으로 한다.

## [발명의 효과]

본 발명에 따르면, 다음과 같은 효과를 달성할 수 있다.

**첫째**, 6계층 PALADIN 방어 아키텍처와 한국어 특화 12개 탐지 모듈을 통해, 종래 시스템에서 방어가 불가능했던 CJK 언어학적 우회 공격(자모 분리, 초성 약어, 고어 활용, 동형 문자, 코드 스위칭 등)을 효과적으로 탐지 및 차단할 수 있다.

**둘째**, 4계층 할루시네이션 검증 파이프라인과 5차원 신뢰도 스코어링을 통해, AI 생성 콘텐츠의 사실적 정확성을 원본 문서 기준으로 다각적으로 검증할 수 있으며, 특히 수치 오류에 대한 문맥 인식 교차검증을 통해 종래 NLI 단독 모델 대비 수치 할루시네이션 탐지율을 현저히 향상시킬 수 있다.

**셋째**, 4대 국제 규제 프레임워크(EU AI Act, K-AI Act, NIST AI RMF, OWASP LLM Top 10)에 대한 자동화된 준수 평가 및 증거 기반 리포트 생성을 통해, 규제 감사(audit) 대응 비용을 대폭 절감하고 AI 거버넌스 체계를 효율적으로 운영할 수 있다.

**넷째**, 보안 방어, 할루시네이션 검증, 규제 준수 평가가 문서 편집 워크플로우에 통합됨으로써, 종래 분절된 도구 사용에 따른 정보 손실 및 업무 비효율을 해소하고, 단일 인터페이스에서 AI 안전 문서 편집의 전체 생명주기를 관리할 수 있다.

**다섯째**, 온라인-오프라인 하이브리드 아키텍처를 통해, 서버 장애 또는 네트워크 단절 시에도 가드레일 규칙 검증 및 수치 교차검증이 클라이언트 측에서 자동으로 수행되어 핵심 안전 기능의 연속성이 보장된다.

## [발명을 실시하기 위한 구체적인 내용]

이하, 첨부된 도면을 참조하여 본 발명의 바람직한 실시예를 상세히 설명한다. 다만, 본 발명의 기술적 사상은 이하의 실시예에 한정되지 않으며, 당해 기술분야의 통상의 지식을 가진 자에 의하여 다양한 변형 실시가 가능함은 물론이다.

### 1. 시스템 전체 구성 (도 1 참조)

본 발명의 일 실시예에 따른 지능형 문서 편집 시스템(100)은 도 1에 도시된 바와 같이, 문서 파싱 및 편집 모듈(110), AI 콘텐츠 생성 모듈(120), 다계층 보안 방어 모듈(130), 할루시네이션 검증 모듈(140), 규제 준수 평가 모듈(150), 및 통합 제어 모듈(160)을 포함하여 구성된다.

상기 각 모듈은 상태 관리 저장소(170)를 통해 데이터를 공유하며, 상기 통합 제어 모듈(160)은 사용자 인터페이스 계층(180)과 상기 각 모듈 사이의 데이터 흐름을 오케스트레이션한다.

#### 1.1 문서 파싱 및 편집 모듈 (110)

문서 파싱 및 편집 모듈(110)은 문서 파서(111), 문서 객체 모델 관리부(112), 인라인 편집기(113), 이력 관리부(114), 및 문서 내보내기부(115)를 포함한다.

**문서 파서(111)**는 HWPX 포맷의 압축 아카이브를 해제하고, 내부 XML 구조(content.xml, docinfo.xml)를 파싱하여 계층적 문서 객체 모델로 변환한다. 상기 문서 객체 모델은 섹션(Section), 단락(Paragraph), 테이블(Table), 이미지(Image), 도형(Shape) 등의 요소를 트리 구조로 표현하며, 각 요소는 서식 속성(폰트, 크기, 색상, 정렬 등)을 포함한다.

또한, 상기 문서 파서(111)는 레거시 HWP 바이너리 포맷에 대하여 OLE(Object Linking and Embedding) 컨테이너를 파싱하고 내부 스트림을 추출하여 HWPX 호환 문서 객체 모델로 변환하는 HWP-HWPX 변환 기능을 포함하며, 텍스트 100%, 문자 서식 95%, 테이블 100%, 이미지 90%의 변환 정확도를 달성한다.

**인라인 편집기(113)**는 셀 단위 및 단락 단위의 실시간 편집을 지원하며, 편집 상태(editingPath, editingContent, originalContent)를 추적하여 문서 객체 모델에 반영한다.

**이력 관리부(114)**는 스택 기반의 실행 취소/재실행(Undo/Redo) 메커니즘을 제공하며, 원자적 연산(atomic operation) 단위로 이력을 관리한다.

**문서 내보내기부(115)**는 편집된 문서 객체 모델을 HWPX, PDF 등의 포맷으로 변환하여 출력한다.

#### 1.2 AI 콘텐츠 생성 모듈 (120) (도 10 참조)

AI 콘텐츠 생성 모듈(120)은 문서 구조 분석부(121), 프롬프트 빌더(122), LLM API 통신부(123), 및 콘텐츠 병합부(124)를 포함한다.

**문서 구조 분석부(121)**는 상기 문서 객체 모델을 입력받아, 문서 유형(월간보고서, 주간보고서, 교안, 회의록 등)을 자동 판별하고, 헤더-콘텐츠 쌍(header-content pair)을 추출하며, 테이블 내 열/행 간의 의미적 관계를 분석한다. 상기 문서 구조 분석은 다음의 단계를 포함한다:

- **단계 S121-1**: 문서 내 반복 패턴(번호 매기기, 날짜 형식, 제목 서식) 탐지
- **단계 S121-2**: 헤더 셀과 데이터 셀의 구분 (서식, 위치, 내용 기반)
- **단계 S121-3**: 테이블 간 교차 참조 관계 분석
- **단계 S121-4**: 문서 유형별 콘텐츠 생성 전략 결정

**프롬프트 빌더(122)**는 상기 분석 결과를 기반으로, 문서 구조를 보존하면서 콘텐츠만 변경하도록 제약된 문맥 인식 프롬프트를 구성한다. 상기 프롬프트는 시스템 메시지(문서 구조 보존 지시), 문서 유형 컨텍스트, 헤더-콘텐츠 매핑, 서식 패턴 샘플을 포함한다.

**LLM API 통신부(123)**는 외부 LLM API(OpenAI GPT-4o, Anthropic Claude, DeepSeek 등)와 통신하며, 재시도 로직(지수 백오프, 최대 3회), 타임아웃 관리(120초), 토큰 사용량 추적 및 비용 관리 기능을 포함한다.

**콘텐츠 병합부(124)**는 LLM이 반환한 구조화된 응답(JSON 형식의 헤더-콘텐츠 매핑)을 상기 문서 객체 모델의 대응 위치에 병합한다. 상기 병합은 원본 문서의 서식 속성(폰트, 크기, 색상, 정렬, 셀 병합 등)을 보존하면서 텍스트 콘텐츠만 치환하는 방식으로 수행된다.

#### 1.3 다계층 보안 방어 모듈 — AEGIS (130) (도 3, 4 참조)

다계층 보안 방어 모듈(130)은 6계층 PALADIN 방어 오케스트레이터를 핵심으로 하며, 각 계층은 순차적으로 실행되어 누적적 위험 점수(risk score)를 산출한다.

##### 1.3.1 제0계층: 신뢰 경계 검증부 (131)

신뢰 경계 검증부(131)는 입력 텍스트에 대한 1차 정제(sanitization)를 수행한다. 구체적으로:
- 유니코드 정규화(NFC/NFKC)를 통한 문자열 표준화
- 제로폭 문자(Zero-Width Characters: U+200B, U+200C, U+200D, U+FEFF) 탐지 및 제거
- 동형 문자(Homoglyph) 탐지: 95개 이상의 혼동 가능 문자쌍 데이터베이스를 참조하여 시각적으로 유사한 문자의 치환 공격을 탐지하고 정규화

##### 1.3.2 제1계층: 의도 분석부 (132)

의도 분석부(132)는 사용자 입력의 의도를 분석하고 기만적 패턴을 탐지한다. 탈옥(Jailbreak) 패턴(역할극 탈출, 제약 조건 제거 시도 등), 프롬프트 인젝션(직접/간접 주입), 사회공학적 공격(신뢰 조작, 권위 남용) 등 10개 카테고리의 공격 패턴을 분석한다.

##### 1.3.3 제2계층: RA 가드 (133)

RA 가드(133)는 검색증강생성(Retrieval-Augmented Generation, RAG) 환경에서의 공격을 방어한다. 검색된 문서에 삽입된 간접 프롬프트 인젝션, 악성 컨텍스트 주입 등을 탐지한다.

##### 1.3.4 제3계층: ClassRAG (134)

ClassRAG(134)는 문맥 기반 분류를 통해 RAG 특유의 위협을 탐지한다. 입력 텍스트의 의미적 분류 결과와 검색된 컨텍스트의 일관성을 검증한다.

##### 1.3.5 제4계층: 회로 차단기 (135)

회로 차단기(135)는 비율 제한(Rate Limiting) 및 임계값 기반 차단 메커니즘을 제공한다. 세션당 요청 빈도, 누적 위험 점수, 연속 의심 요청 횟수 등을 모니터링하여 임계값 초과 시 자동 차단한다.

##### 1.3.6 제5계층: 행동 분석부 (136)

행동 분석부(136)는 다회차 대화(Multi-turn Conversation)에서의 행동 패턴을 분석하여 크레센도 공격(Crescendo Attack)을 탐지한다. 크레센도 공격이란, 단일 회차에서는 무해하게 보이나 복수 회차에 걸쳐 점진적으로 악의적 의도를 에스컬레이션하는 공격 기법으로, 본 발명에서는 다음의 3가지 패턴을 탐지한다:

- **패턴 A (일반→전문→구체)**: 일반적 질문에서 시작하여 전문 용어 사용, 최종적으로 구체적 기법 요청
- **패턴 B (교육→심화→유해)**: 교육적 맥락에서 시작하여 점진적 심화, 최종적으로 유해 정보 요청
- **패턴 C (신뢰→역할→제약해제)**: 신뢰 구축 대화에서 시작하여 역할 부여, 최종적으로 제약 조건 해제 시도

각 대화 회차의 위험도는 세션 기반으로 누적되며, 감쇠 함수(decay function)를 적용하여 시간 경과에 따른 위험도 자연 감쇠를 반영한다.

##### 1.3.7 한국어 언어학적 공격 탐지 서브모듈 (137) (도 4 참조)

상기 각 계층에 횡단적으로 적용되는 한국어 특화 탐지 서브모듈(137)은 12개의 독립 탐지기(Detector)로 구성되며, 각 탐지기는 한국어 고유의 언어학적 특성을 악용한 우회 공격을 탐지한다:

| 서브모듈 번호 | 탐지기 명칭 | 탐지 대상 |
|:---:|---|---|
| 137-1 | 키보드 매핑 탐지기 | 한국어 자판 배열을 이용한 인코딩 공격 |
| 137-2 | 고어(古語) 탐지기 | 고대 한국어 문자를 이용한 필터 우회 |
| 137-3 | 동형 문자 탐지기 | 시각적으로 동일한 한국어 문자 치환 |
| 137-4 | 한자 탐지기 | 한자(漢字) 주입을 통한 우회 |
| 137-5 | 토크나이저 취약점 탐지기 | 토큰 경계를 악용한 우회 |
| 137-6 | 자모 정규화기 | 한글 자모 분해(ㄱ+ㅏ=가)를 이용한 우회 |
| 137-7 | 은어 탐지기 | 한국어 은어/속어를 이용한 유해 표현 |
| 137-8 | 조사 탐지기 | 한국어 형태소 조사를 이용한 의미 변조 |
| 137-9 | 음절 역순 탐지기 | 문자 순서 역전을 통한 우회 |
| 137-10 | 음운 변이 탐지기 | 유사 발음 대체를 통한 우회 |
| 137-11 | 초성 디코더 | 초성 약어(ㅈㅅ → 죄송) 기반 유해 표현 |
| 137-12 | 코드 스위칭 탐지기 | 한국어-영어 등 혼합 언어 공격 |

상기 각 탐지기는 29개의 유해 키워드 목록(폭탄, 마약, 해킹, 테러 등)과 연동되어, 정규화된 텍스트에서의 유해 표현 포함 여부를 판별한다.

##### 1.3.8 결정론적 방어 관리자 (θ=0 보장)

상기 각 계층은 결정론적 방어 관리자(Deterministic Defense Manager)를 포함하며, 기지의(known) 위험 패턴에 대해서는 확률적 판단을 배제하고 100% 신뢰도(θ=0)로 차단하는 것을 보장한다. 이는 종래의 확률적 모델이 가지는 우회 가능성(false negative)을 원천적으로 차단하기 위한 것이다.

##### 1.3.9 위험 점수 산출

최종 위험 점수는 다음의 수식에 의해 산출된다:

```
rawScore = min(layerScore + koreanScore + piiScore, 100)
finalScore = min(rawScore × sensitivity, 100)
```

여기서 `layerScore`는 6개 방어 계층의 누적 점수, `koreanScore`는 한국어 탐지 서브모듈의 점수, `piiScore`는 개인정보 탐지 점수, `sensitivity`는 사용자 설정 감도 계수(0.5~2.0)이다. 최종 위험 수준은 critical(60+), high(40~60), medium(20~40), low(<20)로 분류되며, 설정된 차단 임계값(기본값 60) 이상인 경우 해당 요청이 차단된다.

##### 1.3.10 개인정보 보호 기능

상기 보안 방어 모듈(130)은 개인정보 보호 기능으로서:
- **PII 스캐너**: 한국 주민등록번호(검증 포함), 신용카드번호(Luhn 체크섬), 이메일, 한국 전화번호, 국제 전화번호, 미국 SSN, 여권번호, IPv4 주소의 8종 개인정보를 탐지
- **PII 프록시 엔진**: 형식 보존 가명처리(Format-Preserving Pseudonymization)를 수행하고, 세션 기반 복원 기능을 제공
- **DLP 게이트웨이**: 구성 가능한 정책에 따른 데이터 유출 방지

를 추가로 포함한다.

#### 1.4 할루시네이션 검증 모듈 — TruthAnchor (140) (도 5, 6 참조)

할루시네이션 검증 모듈(140)은 주장 추출부(141), 증거 매칭부(142), 4계층 검증 파이프라인(143), 다차원 스코어링부(144), 및 교정 생성부(145)를 포함한다.

##### 1.4.1 주장 추출부 (141)

주장 추출부(141)는 AI가 생성한 텍스트에서 검증 가능한 개별 주장(claim)을 추출한다. 한국어 특화 문장 분리(종결어미: -다, -습니다, -합니다, -니까 등 인식) 및 복합문 분할(접속 조사: -고, -며, -으며, 쉼표 기준)을 수행하며, 비사실적 표현(인사, 질문, 명령문, 주관적 의견)은 필터링한다. 10자 미만의 주장은 검증 대상에서 제외한다.

##### 1.4.2 증거 매칭부 (142)

증거 매칭부(142)는 벡터 데이터베이스를 사용하지 않는 인메모리 문자열 유사도 기반 증거 매칭을 수행한다. 매칭 점수는 3요소 가중 합산으로 산출된다:
- 키워드 겹침(50%): 불용어 제거 후 의미어 교집합
- 문자 n-그램(30%): 2-그램 겹침률 (패러프레이징 강건성 확보)
- 시퀀스 유사도(20%): 동적 프로그래밍 기반 순서 유사도

상위 k개(문서 크기에 따라 3~7개 동적 조정) 증거 후보를 선정한다.

##### 1.4.3 4계층 검증 파이프라인 (143)

**제0계층 — 가드레일 검증부 (143-0)**

38개의 도메인 규칙으로 구성되며, 규제 위반(투자 조언, 개인정보 노출 등)을 탐지한다. 규칙은 공통(CG-001~010), 금융 특화(CG-018~025), 코드 구조(CG-026~028)의 3개 그룹으로 분류된다. 심각도는 CRITICAL, HIGH, MEDIUM의 3단계로 구분되며, 심각도에 따라 후속 NLI 검증의 임계값이 조정된다.

**제0.5계층 — 수치 교차검증부 (143-1)**

AI 생성 텍스트와 원본 문서에서 수치를 추출하고 교차 비교한다. 인식 가능한 수치 유형은: 백분율(3.5%), 통화(1.7조원), 건수(950만명), 연도, 날짜이다. 각 수치는 문맥 키워드(GDP, 금리, 인구, 고용률, 가격, 자산, 환율 등)와 연동되어 카테고리가 할당되며, **동일 카테고리 내에서만 비교**가 수행된다(예: 금리와 인구수는 비교하지 않음). 허용 오차는 상대오차 5%(백분율은 ±0.5%, 연도는 정확 일치)이다.

**제1계층 — NLI 검증부 (143-2)**

사전학습된 자연어 추론(Natural Language Inference) 모델(DeBERTa-v3-base cross-encoder)을 사용하여 각 주장-증거 쌍에 대한 의미적 관계를 판정한다. 모델의 3-클래스 로짓(contradiction, entailment, neutral)에 소프트맥스 정규화를 적용하여 확률 분포로 변환하며, entailment > 0.55이면 지지, contradiction > 0.65이면 반박, 그 외는 중립으로 판정한다. 단일 증거의 확신도가 낮은 경우, 복수 증거에 대한 앙상블 투표(ensemble voting)를 수행하여 판정 강건성을 확보한다.

NLI 모델을 사용할 수 없는 환경에서는 키워드 겹침률 + 문자 n-그램 유사도를 조합한 폴백(fallback) 메커니즘이 작동한다.

**제2계층 — LLM 재검증부 (143-3)**

제1계층에서 중립(neutral)으로 판정된 주장에 대해 LLM(GPT-4 등)을 호출하여 재검증한다. 주장, 원본 증거, 문맥을 포함한 구조화된 프롬프트를 구성하며, temperature 0.1의 저변동 설정으로 사실 중심의 판정을 유도한다.

##### 1.4.4 5차원 할루시네이션 신뢰도 스코어링 (144) (도 6 참조)

다차원 스코어링부(144)는 검증 결과를 5개 차원에서 평가하여 종합 신뢰도 점수를 산출한다:

| 차원 | 가중치 | 산출 수식 |
|:---:|:---:|---|
| 사실 정확도 (Factual Accuracy) | 35% | 지지 주장 수 / 전체 주장 수 |
| 수치 정확도 (Numerical Accuracy) | 15% | 수치 지지 건수 / 수치 전체 건수 |
| 증거 신뢰도 (Evidence Reliability) | 20% | 신뢰도 가중 평균 (중립 패널티 적용) |
| 일관성 (Consistency) | 15% | 1.0 − (모순비율 × 0.5), 의미 유사도 기반 |
| 불확실성 보정 (Uncertainty Calibration) | 15% | (확정 + 정보중립 × 0.5) / 전체 |

여기서 "정보 중립(informed neutral)"은 신뢰도 > 0.5이고 일부 증거가 있는 중립 주장을, "비정보 중립(uninformed neutral)"은 뒷받침 증거가 없는 중립 주장을 의미하며, 양자를 구분함으로써 불확실성의 질적 차이를 반영한다.

##### 1.4.5 교정 생성부 (145)

반박(contradicted)으로 판정된 주장에 대해, 원본 증거를 기반으로 사실에 부합하는 대안 텍스트를 LLM을 통해 생성한다. temperature 0.1으로 설정하여 창의적 변형을 최소화하고 사실 기반 교정에 집중한다.

#### 1.5 규제 준수 평가 모듈 (150) (도 7 참조)

규제 준수 평가 모듈(150)은 활동 로그 관리부(151), 프레임워크 규칙 엔진(152), 평가 수행부(153), 및 리포트 생성부(154)를 포함한다.

##### 1.5.1 활동 로그 관리부 (151)

상기 AI 콘텐츠 생성 모듈(120), 보안 방어 모듈(130), 할루시네이션 검증 모듈(140)의 모든 동작을 구조화된 활동 로그(AIActivityLog)로 기록한다. 각 로그 엔트리는 다음을 포함한다:

- **동작 메타데이터**: 동작 유형(생성/편집/교정/제안/요약), 트리거(사용자 요청/자동 제안/배치)
- **모델 정보**: 모델명, 공급자, 버전, temperature, maxTokens
- **입출력 데이터**: 입력 토큰 수, 출력 토큰 수, 프롬프트 해시, 생성 텍스트
- **인간 감독 기록**: 사용자 검토 여부, 승인 여부, 수정 여부
- **검증 결과**: 할루시네이션 검증 수행 여부, 점수, 도메인, 주장별 결과
- **보안 결과**: 프롬프트 인젝션 스캔 결과, 출력 정제 여부, PII 탐지 및 조치

최근 500건의 로그가 로컬 저장소에 영속화되며, 세션 기반 그룹화를 통해 관련 동작의 추적성을 확보한다.

##### 1.5.2 프레임워크 규칙 엔진 (152)

4개 규제 프레임워크에 대한 선언적 평가 규칙을 관리한다:

**EU AI Act (Regulation 2024/1689)**: 투명성, 위험관리, 데이터 거버넌스, 인간 감독, 기술적 강건성의 5개 카테고리, 15개 이상의 평가 항목

**한국 AI 기본법 (2026년 시행)**: 안전성, 투명성, 공정성, 책임성, 프라이버시의 5개 카테고리

**NIST AI RMF 1.0**: Govern, Map, Measure, Manage의 4개 기둥(Pillar), 12개 이상의 평가 항목

**OWASP LLM Top 10 (v1.1, 2024)**: 프롬프트 인젝션, 불안전 출력, 모델 DoS, 민감정보 노출, 과도한 의존 등 10개 카테고리, 20개 이상의 평가 항목

총 75개 이상의 평가 규칙이 선언적으로 정의되며, 각 규칙은 구성(configuration) 검사, 활동(activity) 기반 검사, 임계값(threshold) 기반 검사의 3가지 유형으로 분류된다.

##### 1.5.3 평가 수행부 (153)

상기 활동 로그와 시스템 구성 정보를 입력으로 받아, 각 프레임워크의 규칙을 순차적으로 적용하여 평가를 수행한다. 각 규칙의 평가 결과는 pass(통과), warn(경고), fail(실패), n/a(해당없음)의 4단계로 산출되며, 각 결과에는 구체적 증거(evidence)가 첨부된다.

점수 산출 알고리즘:
- 항목별 점수: pass=100, warn=60, fail=0
- 카테고리 점수: 해당 카테고리 내 적용 가능 항목의 평균
- 전체 점수: 모든 카테고리 점수의 평균
- 준수 상태: ≥80 준수(compliant), 50~79 부분 준수(partial), <50 미준수(non-compliant)

##### 1.5.4 리포트 생성부 (154)

평가 결과를 시각화된 컴플라이언스 리포트로 생성한다. 상기 리포트는:
- 프레임워크별 전체 점수 및 준수 상태
- 카테고리별 세부 점수
- 개별 항목의 pass/warn/fail 상태, 증거, 교정 지침
- AI 사용 요약(총 동작 수, 사용 모델, 승인율, 평균 할루시네이션 점수)
- 권고 사항 목록

을 포함하며, PDF 및 JSON 형식으로 내보내기가 가능하다.

#### 1.6 통합 제어 모듈 (160) (도 2, 9 참조)

통합 제어 모듈(160)은 사용자의 AI 콘텐츠 생성 요청 시, 다음의 파이프라인을 자동으로 오케스트레이션한다:

**단계 S1 — 입력 보안 검증**: 사용자의 프롬프트를 보안 방어 모듈(130)에 전달하여 입력 측 보안 검증을 수행한다. PII가 탐지된 경우 가명처리를 수행한다.

**단계 S2 — AI 콘텐츠 생성**: 보안 검증을 통과한 입력을 AI 콘텐츠 생성 모듈(120)에 전달하여 문서 구조 분석 및 LLM 기반 콘텐츠 생성을 수행한다.

**단계 S3 — 출력 보안 검증**: 생성된 콘텐츠를 보안 방어 모듈(130)에 재전달하여 출력 측 보안 검증을 수행한다. 유해 콘텐츠, PII 노출, 정보 유출 등을 검사한다.

**단계 S4 — 할루시네이션 검증**: 보안 검증을 통과한 콘텐츠를 할루시네이션 검증 모듈(140)에 전달하여 원본 문서 기준의 사실 정확성 검증을 수행한다.

**단계 S5 — 결과 제시 및 인간 감독**: 검증 결과(할루시네이션 점수, 반박 주장, 교정 제안)를 사용자에게 제시하고, 사용자의 검토/승인/수정 입력을 수신한다.

**단계 S6 — 문서 반영**: 사용자가 승인한 콘텐츠를 문서 객체 모델에 반영한다.

**단계 S7 — 컴플라이언스 로깅**: 상기 전체 과정의 세부 사항을 활동 로그로 기록하여 규제 준수 평가의 근거로 활용한다.

#### 1.7 온라인-오프라인 하이브리드 아키텍처 (도 8 참조)

본 발명의 시스템은 서버 가용성에 따라 자동으로 운용 모드를 전환하는 하이브리드 아키텍처를 채택한다.

**온라인 모드**: 전체 4계층 검증 파이프라인(가드레일 → 수치검증 → NLI → LLM 재검증)이 서버 측에서 실행된다. NLI 모델(DeBERTa-v3-base)은 최초 요청 시 지연 로딩(lazy loading)되며, 이후 요청에서는 인메모리 상태를 재활용한다.

**오프라인 모드**: 서버 접속 불가 시, 클라이언트 측 JavaScript 엔진이 자동으로 활성화되어 가드레일 규칙 검증 및 수치 교차검증을 수행한다. NLI 및 LLM 기반 검증은 생략되나, 핵심 규제 위반 탐지 및 수치 할루시네이션 탐지는 지속된다.

**모드 전환**: 서버 상태를 주기적으로 감시(health check)하며, 서버 복구 시 자동으로 온라인 모드로 전환한다. 오프라인 모드에서 축적된 미검증 항목은 온라인 전환 시 일괄 재검증이 가능하다.

### 2. 상태 관리 저장소 (170)

상태 관리 저장소(170)는 7개의 독립적 상태 저장소(Store)로 구성되며, 각 저장소는 반응형 상태 관리 라이브러리(Zustand)를 통해 구현된다:

| 저장소 | 관리 대상 |
|---|---|
| 문서 저장소 (documentStore) | 문서 데이터, 파일명, 변경 상태 |
| 편집 저장소 (editingStore) | 현재 편집 경로, 편집 내용, 원본 내용 |
| AI 저장소 (aiStore) | API 키, 처리 상태, 메시지 이력 |
| UI 저장소 (uiStore) | 테마, 사이드바, 알림 |
| 컴플라이언스 저장소 (complianceStore) | 활동 로그, 평가 보고서, 세션 ID |
| 셀 선택 저장소 (cellSelectionStore) | 선택 셀, 선택 모드 |
| 템플릿 저장소 (templateStore) | 문서 템플릿, 선택된 템플릿 |

각 저장소는 미들웨어를 통해 개발자 도구 연동(devtools) 및 로컬 저장소 영속화(persist)를 지원한다.

## [부록: 기술 용어 정의]

| 용어 | 정의 |
|---|---|
| **LLM (Large Language Model)** | 대규모 텍스트 데이터로 사전학습된 자연어 처리 AI 모델 |
| **할루시네이션 (Hallucination)** | LLM이 사실과 다른 정보를 생성하는 현상 |
| **프롬프트 인젝션 (Prompt Injection)** | LLM의 시스템 지시를 조작하여 의도하지 않은 동작을 유도하는 공격 |
| **탈옥 (Jailbreak)** | LLM의 안전 제약을 우회하여 금지된 응답을 유도하는 공격 |
| **NLI (Natural Language Inference)** | 두 문장 간의 논리적 관계(함의/모순/중립)를 판별하는 자연어 처리 과제 |
| **RAG (Retrieval-Augmented Generation)** | 외부 문서 검색을 통해 LLM 생성의 정확도를 향상시키는 기법 |
| **크레센도 공격 (Crescendo Attack)** | 다회차 대화에서 점진적으로 악의적 의도를 에스컬레이션하는 공격 기법 |
| **PII (Personally Identifiable Information)** | 개인을 식별할 수 있는 정보 (주민등록번호, 전화번호 등) |
| **DLP (Data Loss Prevention)** | 민감 데이터의 외부 유출을 방지하는 보안 기술 |
| **HWPX** | 한글과컴퓨터의 개방형 문서 포맷 (OWPML 기반 XML 구조) |
| **PALADIN** | 본 발명의 6계층 순차적 방어 오케스트레이션 아키텍처의 명칭 |
| **θ=0 보장** | 기지의 위험 패턴에 대해 확률적 판단을 배제하고 100% 차단을 보장하는 결정론적 방어 메커니즘 |
