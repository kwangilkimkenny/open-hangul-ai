## 개발 배경

### 웹 기반 한글 문서 처리의 현실적 한계

현재 국내 공공기관과 기업에서 광범위하게 사용되는 HWPX(한컴오피스 문서) 파일을
웹 환경에서 처리하는 것은 기술적으로 복잡한 과제다. 기존 솔루션들은 서버 기반
변환에 의존하거나, 데스크톱 소프트웨어 설치를 요구하는 한계가 있었다.

YATAV는 이러한 기술적 공백을 인식하고, **브라우저에서 직접 HWPX 파일을 처리할 수
있는 오픈소스 라이브러리** 개발에 착수했다고 발표했다.

## 프로젝트 개요

### 오픈한글AI: 신뢰할 수 있는 한글 문서 편집 솔루션

**오픈한글AI**는 React 및 TypeScript 기반의 웹 라이브러리로, 다음과 같은 핵심
목표를 설정하고 개발을 진행하고 있다:

#### 1. 기술적 목표

- **HWPX 네이티브 지원**: 서버 변환 없이 브라우저에서 직접 HWPX 파일 파싱 및
  렌더링
- **React 생태계 통합**: npm 패키지로 제공하여 React 프로젝트에 즉시 통합 가능
- **TypeScript 완전 지원**: 타입 안정성과 개발자 경험 최적화

#### 2. 사용자 경험 목표

- **5분 설치**: `npm install open-hangul-ai` 명령어만으로 즉시 사용 가능
- **직관적 API**: 복잡한 설정 없이 `<HWPXViewer />` 컴포넌트로 간단 구현
- **반응형 디자인**: 데스크톱부터 모바일까지 일관된 사용자 경험

## 기술적 차별화 요소

### 1. 브라우저 네이티브 처리

기존 솔루션들이 서버 기반 변환에 의존하는 반면, 오픈한글AI는 **클라이언트
사이드에서 직접 HWPX 파일을 처리**한다. 이를 통해:

- 서버 부하 제거
- 실시간 처리 가능
- 개인정보 보호 강화 (파일이 서버로 전송되지 않음)

### 2. 개발자 친화적 설계

React 생태계의 표준을 따라 개발자가 직관적으로 사용할 수 있도록 설계:

```javascript
import { HWPXViewer } from 'open-hangul-ai';

function App() {
  return <HWPXViewer fileUrl="/문서.hwpx" />;
}
```

### 3. 오픈소스 접근법

MIT 라이센스로 제공하여 상업적 사용을 포함한 자유로운 활용이 가능하며, 커뮤니티
기여를 통한 지속적 발전을 추진한다.
