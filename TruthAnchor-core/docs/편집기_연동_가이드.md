## 1. 개요

### 1.1 목적

편집기의 AI 어시스턴트가 생성한 텍스트에 **할루시네이션(사실과 다른 정보)**이 포함되었는지 자동 검증하고, 원본 문서에 기반한 수정문을 제안합니다.

### 1.2 검증 방식

편집기의 기존 "TruthAnchor" 플레이스홀더를 실제 구현으로 대체합니다.

- **트리거**: 사용자가 검증 버튼 클릭 (배치 검증)
- **입력**: 원본 문서 텍스트 + AI 생성 텍스트
- **출력**: 클레임별 지지/모순/미확인 판정 + 5차원 신뢰도 스코어 + 수정문

### 1.3 4계층 검증 파이프라인

```
Layer 0:   가드레일 38규칙    (<1ms)  → 투자 권유, PII, 규제 위반 즉시 탐지
Layer 0.5: 수치 교차검증      (<1ms)  → 금리, 금액, 날짜 등 수치 불일치
Layer 1:   NLI 의미적 검증   (~50ms)  → DeBERTa 모델로 의미적 모순 탐지
Layer 2:   LLM 재검증         (~2s)   → GPT-4로 미확인 클레임 재판정 (선택)
```

## 2. 아키텍처

### 2.1 연동 구조

```
│  │  (HWP/MD)    │    │  (GPT-4)     │    │  (HalluGuard) │  │
│  └──────┬───────┘    └──────┬───────┘    └──────┬────────┘  │
│         │                   │                   │            │
└─────────┼───────────────────┼───────────────────┼────────────┘
          │                   │                   │
          │     ┌─────────────┤                   │
          │     ▼             │                   ▼
          │  ┌──────────┐    │          ┌─────────────────┐
          │  │ OpenAI   │    │          │ HalluGuard Core │
          │  │ API      │◀───┘          │ :8200           │
          │  └──────────┘               │                 │
          │                             │ 가드레일 38규칙  │
          │                             │ 수치 교차검증    │
          │                             │ NLI (DeBERTa)   │
          │                             │ LLM (GPT-4공유) │
          │                             └─────────────────┘
          │
     ┌────┴─────┐
     │IndexedDB │  (원본 문서 자동 저장)
     │localStorage│
     └──────────┘
```

### 2.2 데이터 흐름

```
1. 사용자가 원본 문서(.hwp/.hwpx/.md)를 편집기에서 열기
2. AI 어시스턴트가 텍스트 생성 (GPT-4)
3. 사용자가 "할루시네이션 검증" 버튼 클릭
4. 프론트엔드가 원본 텍스트 + AI 생성 텍스트를 HalluGuard API로 전송
5. HalluGuard가 4계층 검증 수행 → 결과 반환
6. 프론트엔드가 클레임별 하이라이팅 + 스코어카드 표시
```

### 2.3 포트 할당

| 서비스 | 포트 | 용도 |
|--------|------|------|
| Vite 개발 서버 | 5090 | 편집기 프론트엔드 |
| Node.js 프록시 | 3000 (또는 Vite 내장) | OpenAI API 프록시 |
| **HalluGuard Core** | **8200** | 할루시네이션 검증 API |

## 3. HalluGuard 서비스 설치 및 실행

### 3.1 방법 A: 직접 실행 (개발 환경)

```bash
# 1. halluguard-core 폴더를 편집기 프로젝트 옆에 배치
# 예: ~/projects/ai-editor/
#     ~/projects/halluguard-core/

# 2. Python 가상환경 생성 및 의존성 설치
cd halluguard-core
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. 환경변수 설정
cp .env.example .env
# .env 파일을 열고 편집기와 동일한 OpenAI API 키 입력:
# OPENAI_API_KEY=sk-...

# 4. 서버 실행
uvicorn api:app --host 0.0.0.0 --port 8200 --reload

# 서버 확인
curl http://localhost:8200/health
```

## 4. 편집기 프록시 설정

### 4.1 Vite 프록시 설정 (권장)

편집기의 `vite.config.ts`에 프록시 규칙을 추가합니다:

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5090,
    proxy: {
      // 기존 OpenAI 프록시 (이미 설정되어 있음)
      '/api/ai/chat': {
        target: 'https://api.openai.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api\/ai\/chat/, '/v1/chat/completions'),
      },

      // ★ HalluGuard 검증 API 추가 — 이 설정만 추가하면 됩니다
      '/api/v2/validate': {
        target: 'http://localhost:8200',
        changeOrigin: true,
      },
      '/health': {
        target: 'http://localhost:8200',
        changeOrigin: true,
      },
    },
  },
});
```

### 4.2 Node.js 프록시 설정 (대안)

기존 `server/proxy.js`를 사용하는 경우, HalluGuard 라우트를 추가합니다:

```javascript
// server/proxy.js에 추가
const HALLUGUARD_URL = process.env.HALLUGUARD_URL || 'http://localhost:8200';

// HalluGuard 프록시 라우트
if (pathname.startsWith('/api/v2/validate')) {
  const targetUrl = `${HALLUGUARD_URL}${pathname}`;
  const proxyReq = http.request(targetUrl, {
    method: req.method,
    headers: { ...req.headers, host: new URL(HALLUGUARD_URL).host },
  }, (proxyRes) => {
    res.writeHead(proxyRes.statusCode, proxyRes.headers);
    proxyRes.pipe(res);
  });
  req.pipe(proxyReq);
  return;
}
```

### 4.3 커스텀 엔드포인트 (직접 호출)

프록시 없이 HalluGuard를 직접 호출할 수도 있습니다:

```typescript
const HALLUGUARD_URL = import.meta.env.VITE_HALLUGUARD_URL || 'http://localhost:8200';
```

이 경우 HalluGuard의 CORS 설정이 이미 `allow_origins=["*"]`이므로 추가 설정이 불필요합니다.

## 5. API 명세

### 5.1 POST `/api/v2/validate` — 단건 검증

**요청:**

```typescript
interface ValidateRequest {
  source_text: string;  // 원본 문서 텍스트 (필수, 1자 이상)
  llm_output: string;   // AI 생성 텍스트 (필수, 1자 이상)
  domain?: string;      // 도메인 (기본: "general")
                        // "general" | "finance" | "defense" | "government" | "education"
}
```

**응답:**

```typescript
interface ValidateResponse {
  overall_score: number;      // 0.0~1.0 (supported / total)
  total_claims: number;       // 추출된 클레임 수
  supported_claims: number;   // 지지된 클레임 수
  contradicted_claims: number; // 모순된 클레임 수
  neutral_claims: number;     // 미확인 클레임 수
  claims: ClaimResult[];      // 개별 클레임 판정 결과
  multi_scores: MultiScores;  // 5차원 신뢰도 스코어
  corrected_text: string;     // 교정된 전체 텍스트
  elapsed_ms: number;         // 처리 시간 (밀리초)
}

interface ClaimResult {
  claim_text: string;              // 원본 클레임 텍스트
  verdict: 'supported' | 'contradicted' | 'neutral';
  confidence: number;              // 0.0~1.0 판정 확신도
  evidence_text: string;           // 판정 근거
  suggested_correction: string | null; // 수정 제안 (contradicted만)
  claim_order: number;             // 클레임 순서 (0-based)
}

interface MultiScores {
  factual_accuracy: number;        // 사실 정확도 (35%)
  numerical_accuracy: number;      // 수치 정확도 (15%)
  evidence_reliability: number;    // 근거 신뢰도 (20%)
  consistency: number;             // 일관성 (15%)
  uncertainty_calibration: number; // 판정 완결도 (15%)
  overall: number;                 // 가중 종합 점수
  details: {
    total_claims: number;
    supported: number;
    contradicted: number;
    neutral: number;
    neutral_informed: number;      // 근거 있는 neutral
    neutral_uninformed: number;    // 근거 없는 neutral
    numerical_claims: number;
    weights: Record<string, number>;
  };
}
```

**verdict 의미:**

| verdict | 색상 | 의미 | 조건 |
|---------|------|------|------|
| `supported` | 초록 | 원본과 일치 | NLI entailment > 0.55 |
| `contradicted` | 빨강 | 원본과 모순 | 가드레일 위반, 수치 불일치, NLI contradiction > 0.65 |
| `neutral` | 노랑 | 판단 불가 | 관련 근거 없거나 확신도 부족 |

## 6. 프론트엔드 연동 코드

### 6.1 API 클라이언트 (`src/lib/ai/halluguard-client.ts`)

```typescript
/**
 * HalluGuard Core API 클라이언트
 * 할루시네이션 검증 서비스와 통신합니다.
 */

const BASE_URL = import.meta.env.VITE_HALLUGUARD_URL || '';

// ── 타입 정의 ────────────────────────────────────

export interface ClaimResult {
  claim_text: string;
  verdict: 'supported' | 'contradicted' | 'neutral';
  confidence: number;
  evidence_text: string;
  suggested_correction: string | null;
  claim_order: number;
}

export interface MultiScores {
  factual_accuracy: number;
  numerical_accuracy: number;
  evidence_reliability: number;
  consistency: number;
  uncertainty_calibration: number;
  overall: number;
  details: Record<string, unknown>;
}

export interface ValidateResponse {
  overall_score: number;
  total_claims: number;
  supported_claims: number;
  contradicted_claims: number;
  neutral_claims: number;
  claims: ClaimResult[];
  multi_scores: MultiScores;
  corrected_text: string;
  elapsed_ms: number;
}

export interface BatchItemResult {
  id: string;
  result: ValidateResponse | null;
  error: string | null;
}

export interface BatchResponse {
  total: number;
  completed: number;
  failed: number;
  results: BatchItemResult[];
}

export interface HealthResponse {
  status: string;
  version: string;
  llm_model: string;
  nli_enabled: boolean;
  llm_verify_enabled: boolean;
  domains: string[];
}

// ── API 함수 ─────────────────────────────────────

export async function checkHealth(): Promise<HealthResponse> {
  const res = await fetch(`${BASE_URL}/health`);
  if (!res.ok) throw new Error(`Health check failed: ${res.statusText}`);
  return res.json();
}

export async function validate(
  sourceText: string,
  llmOutput: string,
  domain: string = 'general',
): Promise<ValidateResponse> {
  const res = await fetch(`${BASE_URL}/api/v2/validate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      source_text: sourceText,
      llm_output: llmOutput,
      domain,
    }),
  });

  if (!res.ok) {
    const err = await res.json().catch(() => ({ detail: res.statusText }));
    throw new Error(err.detail || `Validation failed: ${res.status}`);
  }

  return res.json();
}

export async function validateBatch(
  items: Array<{ sourceText: string; llmOutput: string; domain?: string }>,
): Promise<BatchResponse> {
  const res = await fetch(`${BASE_URL}/api/v2/validate/batch`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      items: items.map((item, i) => ({
        id: `item-${i}`,
        source_text: item.sourceText,
        llm_output: item.llmOutput,
        domain: item.domain || 'general',
      })),
    }),
  });

  if (!res.ok) throw new Error(`Batch validation failed: ${res.statusText}`);
  return res.json();
}
```

### 6.2 검증 상태 관리 (`src/stores/halluguardStore.ts`)

```typescript
import { create } from 'zustand';
import type { ValidateResponse, ClaimResult } from '@/lib/ai/halluguard-client';

interface HalluGuardState {
  // 상태
  isValidating: boolean;
  result: ValidateResponse | null;
  error: string | null;

  // 액션
  setValidating: (v: boolean) => void;
  setResult: (r: ValidateResponse | null) => void;
  setError: (e: string | null) => void;
  reset: () => void;
}

export const useHalluGuardStore = create<HalluGuardState>((set) => ({
  isValidating: false,
  result: null,
  error: null,

  setValidating: (v) => set({ isValidating: v }),
  setResult: (r) => set({ result: r, error: null }),
  setError: (e) => set({ error: e, isValidating: false }),
  reset: () => set({ isValidating: false, result: null, error: null }),
}));
```

### 6.3 검증 실행 훅 (`src/hooks/useHalluGuard.ts`)

```typescript
import { useCallback } from 'react';
import { validate } from '@/lib/ai/halluguard-client';
import { useHalluGuardStore } from '@/stores/halluguardStore';

export function useHalluGuard() {
  const store = useHalluGuardStore();

  const runValidation = useCallback(
    async (sourceText: string, llmOutput: string, domain: string = 'general') => {
      store.setValidating(true);
      store.setError(null);

      try {
        const result = await validate(sourceText, llmOutput, domain);
        store.setResult(result);
        return result;
      } catch (err) {
        const message = err instanceof Error ? err.message : 'Validation failed';
        store.setError(message);
        return null;
      } finally {
        store.setValidating(false);
      }
    },
    [],
  );

  return {
    ...store,
    runValidation,
  };
}
```

## 7. UI 통합 패턴

### 7.1 검증 버튼 통합

기존 편집기의 `ai-ast-local-check` 버튼 옆에 HalluGuard 검증 버튼을 추가합니다:

```tsx
// 검증 버튼 컴포넌트
function HalluGuardButton() {
  const { isValidating, runValidation } = useHalluGuard();

  const handleClick = async () => {
    // 편집기 상태에서 원본 문서와 AI 생성 텍스트 추출
    const sourceText = getOriginalDocumentText(); // IndexedDB 또는 Zustand에서
    const llmOutput = getCurrentEditorContent();  // 편집기 현재 내용

    await runValidation(sourceText, llmOutput, 'general');
  };

  return (
    <button onClick={handleClick} disabled={isValidating}>
      {isValidating ? '검증 중...' : '할루시네이션 검증'}
    </button>
  );
}
```

### 7.3 5차원 스코어카드

```tsx
function ScoreCard({ scores }: { scores: MultiScores }) {
  const dimensions = [
    { key: 'factual_accuracy', label: '사실 정확도', weight: '35%' },
    { key: 'numerical_accuracy', label: '수치 정확도', weight: '15%' },
    { key: 'evidence_reliability', label: '근거 신뢰도', weight: '20%' },
    { key: 'consistency', label: '일관성', weight: '15%' },
    { key: 'uncertainty_calibration', label: '판정 완결도', weight: '15%' },
  ];

  return (
    <div>
      <h3>신뢰도 스코어: {(scores.overall * 100).toFixed(1)}점</h3>
      {dimensions.map(({ key, label, weight }) => (
        <div key={key} style={{ display: 'flex', alignItems: 'center', gap: 8, margin: '4px 0' }}>
          <span style={{ width: 100 }}>{label}</span>
          <div style={{ flex: 1, background: '#e5e7eb', borderRadius: 4, height: 16 }}>
            <div
              style={{
                width: `${(scores[key as keyof MultiScores] as number) * 100}%`,
                background: '#3b82f6',
                borderRadius: 4,
                height: '100%',
              }}
            />
          </div>
          <span style={{ width: 60 }}>
            {((scores[key as keyof MultiScores] as number) * 100).toFixed(0)}% ({weight})
          </span>
        </div>
      ))}
    </div>
  );
}
```

```env
# 기존 편집기 설정 (변경 없음)

```

### 8.3 API 키 공유 방식

편집기가 이미 사용 중인 OpenAI API 키를 HalluGuard와 공유합니다:

```
편집기 .env                     HalluGuard .env
────────────                    ────────────
VITE_OPENAI_API_KEY=sk-xxx      OPENAI_API_KEY=sk-xxx  ← 동일한 키
```

### 8.4 LLM 없이 동작 (경량 모드)

OpenAI API 키 없이도 동작합니다. 이 경우:
- 가드레일 38규칙: **정상 동작**
- 수치 교차검증: **정상 동작**
- NLI 의미적 검증: **정상 동작** (로컬 DeBERTa 모델)
- LLM 재검증: **비활성화** (neutral 클레임 유지)
- 수정문 생성: **비활성화**

## 10. 트러블슈팅

### 10.1 "Connection refused" — 서버 미실행

```
증상: fetch('/api/v2/validate') → ERR_CONNECTION_REFUSED
원인: HalluGuard 서비스가 실행되지 않음
해결: uvicorn api:app --port 8200 또는 docker compose up -d
```

### 10.2 "422 Validation Error" — 입력 형식 오류

```
증상: HTTP 422 응답
원인: source_text 또는 llm_output이 빈 문자열
해결: 두 필드 모두 1자 이상의 텍스트 필요
```

### 10.3 NLI 모델 로드 실패

```
증상: NLI model load failed, using fallback
원인: sentence-transformers 미설치 또는 모델 다운로드 실패
해결: pip install sentence-transformers torch
영향: 키워드 기반 폴백으로 동작 (정확도 약간 저하)
```

### 10.4 LLM 재검증 비활성화

```
증상: health에서 llm_verify_enabled=false
원인: OPENAI_API_KEY 미설정
해결: .env에 OPENAI_API_KEY 설정
영향: 가드레일 + 수치검증 + NLI는 정상 동작, neutral 클레임 재검증만 비활성
```

### 10.5 CORS 오류

```
증상: Access-Control-Allow-Origin 오류
원인: 직접 호출 시 CORS
해결: HalluGuard는 기본적으로 allow_origins=["*"]이므로 CORS 문제 없음
      Vite 프록시를 통해 호출하면 CORS 자체가 발생하지 않음 (권장)
```

### 10.6 높은 neutral 비율

```
증상: 대부분의 클레임이 neutral로 판정
원인: 원본 문서(source_text)에 관련 내용이 부족
해결: source_text에 AI가 참고해야 할 충분한 원본 내용을 포함
      domain 파라미터를 적절히 설정 (예: "finance")
```
