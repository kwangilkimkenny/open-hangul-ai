## 야타브, TruthAnchor·AEGIS 결합한 '오픈한글AI' 출시

### AI 기본법 시대, 한글 문서 편집에 보안·검증·컴플라이언스를 통합한 국내 최초 올인원 AI 문서 편집 환경

---

### 1. 출시 배경 — "잘 써주는 AI"에서 "믿고 쓸 수 있는 AI 편집 환경"으로

AI를 활용한 문서 작성과 편집이 빠르게 확산되면서, 기업과 공공부문에서는 생산성
향상과 함께 새로운 과제가 동시에 떠오르고 있다. 생성형 AI의
**할루시네이션(환각)**, **개인정보 및 보안 위험**, 그리고 2026년 시행된 **AI
기본법**에 따른 투명성·안전성·신뢰성 확보 의무가 그것이다.

문서 작성은 단순한 텍스트 생성에 그치지 않는다. 보고, 정책 검토, 민원 대응, 내부
커뮤니케이션 등 **실제 의사결정과 직결**되는 만큼, "잘 써주는 AI"를 넘어 **"믿고
쓸 수 있는 AI 편집 환경"**에 대한 요구가 커지고 있다.

이 같은 흐름 속에서 야타브(YATAV)는 대규모 언어모델(LLM)을 활용한 콘텐츠 자동
생성에 더해, **다계층 보안 방어**, **할루시네이션 검증**, **다중 규제 프레임워크
준수 평가**를 하나의 문서 편집 환경 안에 결합한 **'오픈한글AI'**를 출시했다.

---

### 2. 제품 개요 — 생성·검증·방어·컴플라이언스, 단일 편집 흐름으로

오픈한글AI는 한글 공문서 표준 포맷(HWPX)을 네이티브로 지원하는 AI 문서 편집
환경이다. 기존 생성형 AI 편집기가 "문장을 대신 써주는" 수준에 머물렀다면,
오픈한글AI는 **편집 과정 전체에 보안·검증·거버넌스를 내장**하여 문서의 생성과
신뢰를 동시에 보장한다.

| 계층              | 기술                                     | 역할                                                |
| ----------------- | ---------------------------------------- | --------------------------------------------------- |
| **생성 계층**     | GPT-4o 기반 구조화 생성                  | 문서 초안, 문장 보완, 형식 수정, 다국어 변환        |
| **보안 계층**     | AEGIS v5.2 (219개 공격 패턴, 6계층 방어) | 프롬프트 인젝션 차단, PII 자동 가명처리, DLP        |
| **신뢰 계층**     | TruthAnchor v2.0 (75개 도메인 가드레일)  | 사실 검증, 수치 교차검증, 인용 가능성 평가          |
| **거버넌스 계층** | 4대 규제 프레임워크 (70개 체크리스트)    | EU AI Act, AI 기본법, NIST AI RMF, OWASP LLM Top 10 |

### 3. 핵심 기능 상세

#### 3-1. AEGIS — AI 보안 게이트웨이 (v5.2)

AEGIS는 문서 작성 과정에서 **입력·출력 양방향 보안**을 담당하는 다계층 방어
시스템이다.

**공격 방어 체계**

- **219개 공격 패턴 데이터베이스**: 프롬프트 인젝션(45), 탈옥 시도(45), 정보
  추출(36), 역할극 조작(30), 사회공학(21), 코드 인젝션(14), 인코딩 우회(15),
  음성/음향 기반(13)
- **PALADIN 6계층 방어 엔진**: 확정적 패턴 매칭 → 신뢰 경계 검증 → 의도 검증 →
  행동 분석 → 서킷 브레이커 → 커스텀 룰
- **한국어 특화 방어 12개 모듈**: 자모 분리 공격, 한글 유니코드
  동형문자(Homoglyph), 한영 코드스위칭 인젝션, 초성 인코딩, 아래아 한글 고어
  문자 우회 등 한국어 환경에서만 발생하는 공격 벡터를 전용 모듈로 방어

**개인정보 자동 보호 파이프라인 (신규)**

- 사용자가 문서 편집 중 입력한 텍스트에서 **주민등록번호, 신용카드번호,
  전화번호, 이메일, 여권번호** 등 8종 PII를 자동 탐지
- LLM 전송 전 **형식 보존 가명처리(Format-Preserving Pseudonymization)**: 예컨대
  `850101-1234567` → `930715-2XXXXXX`로 변환하여 원본 개인정보가 외부 AI 서버에
  전혀 노출되지 않음
- LLM 응답 수신 후 **세션 기반 자동 복원**: 사용자에게는 원본 정보가 정확히 표시
- **DLP(Data Loss Prevention) 게이트웨이**: AWS 자격증명, 데이터베이스 접속 URI,
  API 키, 개인키, JWT 토큰 등 10종 이상의 민감 데이터 유출 차단

> **"사용자는 평소처럼 문서를 작성하기만 하면 된다. 개인정보 보호는 시스템이
> 자동으로 처리한다."**

**출력 필터링**

- AI 생성 응답에서 자격증명 노출, 유해 콘텐츠, 코드 인젝션 패턴을 실시간
  탐지·차단
- 스트리밍 응답 환경에서도 실시간 필터링 지원

#### 3-2. TruthAnchor — 할루시네이션 검증 엔진 (v2.0)

TruthAnchor는 AI가 생성한 문장의 **사실성, 수치 정확도, 인용 가능성, 응답
일관성**을 검증하는 신뢰 계층이다.

**6대 전문 도메인 검증**

| 도메인              | 가드레일 수 | 주요 검증 항목                                 |
| ------------------- | ----------- | ---------------------------------------------- |
| **일반(General)**   | 18          | 투자 권유 금지, 수익 보장 금지, 미래 예측 제한 |
| **금융(Finance)**   | 8           | 금리 차별, 상품 비교, 수신/대출금리 구분       |
| **의료(Medical)**   | 15          | 투약 권유 금지, 부작용 부정 금지, 처방전 요구  |
| **교육(Education)** | 10          | 졸업 보장 금지, 등록금 혜택 과장 금지          |
| **국방(Defense)**   | 11          | 기밀 정보 노출, 군사 시설 위치, 무기 사양      |
| **행정(Admin)**     | 13          | 법률/조례 정확성, 예산·인구·투표 통계 검증     |

**정밀 수치 교차검증**

- **13종 수치 패턴 인식**: 퍼센트(%p vs %), 조/억/만 단위, 의료 단위(mmHg, mg,
  mL), 날짜, 영문 대수(trillion, billion) 등
- **95개 의미 맥락 패턴**: "수신금리 vs 대출금리 vs 금리차"처럼 같은 숫자라도
  맥락에 따라 다르게 검증
- **단위 호환성 그룹**: %와 %p, 만원과 억원 등 비호환 단위 간 오비교 방지
- **오탐 방지(Oops Prevention)**: %와 %p 자동 구분, 보고 기간과 발행일 구분,
  범위 한정어("모든" vs "일부") 불일치 탐지

**검증 결과 시각화**

- 문장별 **신뢰도 컬러 코딩**: 지지(녹색), 모순(적색), 중립(황색)
- 근거 문서 원문 인용 패널
- 원클릭 교정 적용 버튼
- 다차원 점수: 사실 정확도, 수치 정확도, 근거 신뢰도 분리 표시

#### 3-3. 4대 규제 프레임워크 컴플라이언스 (70개 체크리스트)

오픈한글AI는 문서 편집 과정에서 발생하는 AI 활동을 자동으로 기록하고, 4대 규제
프레임워크에 대한 **자동 준수 평가 리포트**를 생성한다.

| 프레임워크                           | 검사 항목 수 | 주요 평가 영역                                               |
| ------------------------------------ | ------------ | ------------------------------------------------------------ |
| **EU AI Act** (Regulation 2024/1689) | 17           | 투명성, 리스크 관리, 데이터 거버넌스, 인간 감독, 기술 건전성 |
| **한국 AI 기본법** (2026년 시행)     | 17           | 안전성, 투명성, 공정성, 책임성, 프라이버시                   |
| **NIST AI RMF** (v1.0)               | 13           | 거버넌스, 매핑, 측정, 관리                                   |
| **OWASP LLM Top 10** (v1.1 2024)     | 23           | 인젝션, 출력 보안, 데이터 유출, DoS, 과의존 등 10대 위협     |

- 프레임워크별 카테고리 점수와 개별 항목 pass/warn/fail 상태 표시
- 미준수 항목에 대한 **구체적 개선 권고(remediation)** 및 **관련 조항 링크**
- 보수적 모노톤 디자인의 **PDF 리포트 자동 생성** — 감사·보고 용도 즉시 활용
  가능

#### 3-4. 한글 문서 편집 기능

오픈한글AI는 AI 보안·검증 기능 외에도 **완전한 한글 문서 편집 환경**을 제공한다.

- **네이티브 HWPX 파싱**: OWPML 기반 55개 이상의 문서 요소 파싱 (표, 이미지,
  머리글/바닥글, 각주/미주, 번호 매기기 등)
- **레거시 HWP 자동 변환**: 바이너리 HWP 파일을 HWPX로 자동 변환하여 편집 가능
- **구조화된 AI 생성**: 문서 구조(표 헤더-셀 쌍)를 분석하여 맥락에 맞는 콘텐츠
  자동 생성
- **12종 AI 퀵 액션**: 요약, 키워드 추출, 이메일 초안, 회의록 변환, 영문 번역,
  간소화, 리뷰 등
- **다중 페이지 일괄 생성**: 페이지별로 다른 콘텐츠를 한 번에 생성하는 배치 모드
- **실행 취소/다시 실행**: 1,000단계 이상 이력 관리, 1ms 미만 응답 속도
- **자동 저장 및 충돌 복구**: 30초 간격 IndexedDB 기반 자동 저장

### 4. AI 기본법 대응 — 왜 문서 편집기에 컴플라이언스가 필요한가

2026년 시행된 **AI 기본법**은 생성형 AI 서비스에 대해 다음을 요구한다:

- AI에 기반해 운용된다는 사실의 **사전 고지**
- 결과물이 AI에 의해 생성되었다는 사실의 **표시**
- 고영향 AI에 해당하는 경우 **위험관리 방안, 설명 가능성, 이용자 보호, 사람의
  관리·감독, 관련 문서 보관** 등 안전성·신뢰성 확보 조치

문서 편집·생성 서비스는 생성형 AI 기반 서비스에 해당할 가능성이 높다. 따라서
단순 기능 경쟁을 넘어 **규제 준수 도구와 내부 통제 체계**를 함께 갖출 필요가
있다.

오픈한글AI는 이에 대응하여:

- 모든 AI 활동에 **AI 생성 표시(마킹)**를 자동 부착
- AI 모델 정보, 토큰 사용량, 비용을 **투명하게 추적**
- 사용자 승인 없이 AI가 문서를 자동 변경하지 않는 **인간 감독 체계**
- 실행 취소/다시 실행을 통한 **롤백 메커니즘**
- AI 활동 로그를 기반으로 **감사 추적(Audit Trail)** 자동 생성

### 5. 행안부 'AI 친화 행정문서' 정책과의 연계

행정안전부는 2026년 3월 **'AI 시대 행정문서 작성 가이드라인'**을 시범 도입하며,
공무원 보고서와 행정문서를 AI가 읽고 활용하기 쉬운 구조로 바꾸는 작업에
착수했다.

행안부의 주요 방향:

- 주어와 서술어를 명확히 하고, 문장 구조를 단순화
- 복잡한 셀 병합을 금지하고 단순 구조의 표 사용 권장
- 자료 요약, 회의록 자동 생성, 보고서 초안 작성, 단순 민원 응답에 AI 활용 확대

윤호중 행정안전부 장관은 "장관에게 올라오는 보고서부터 가이드라인에 따라 형식에
얽매이지 않고 작성하도록 해달라"며 보고서의 겉치레보다 **내용과 현장의 목소리에
집중**하겠다고 밝혔다.

오픈한글AI는 이 정책 방향과 정확히 맞닿는다:

- **HWPX 네이티브 지원**으로 행정문서 표준 포맷 완벽 호환
- **구조 분석 기반 AI 생성**으로 문서의 논리 구조를 파악한 뒤 맥락에 맞는 콘텐츠
  제안
- **표 구조 분석기**로 복잡한 병합 셀 대신 단순 구조 권장
- **AI 퀵 액션**으로 요약, 회의록 변환, 보고서 초안 등 행안부 권장 활용 사례
  즉시 지원

> **문서는 이제 사람이 보기 좋게만 만드는 형식물이 아니라, 사람과 AI가 함께 읽고
> 이해하며 활용할 수 있는 업무 인프라다.**

### 6. 기술 아키텍처 — 생성과 검증의 자동화된 파이프라인

```
┌─────────────────────────────────────────────────────────────────┐
│                    사용자 문서 편집 환경                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [1] 입력 보안 ──────────────────────────── AEGIS v5.2          │
│      ├─ 프롬프트 인젝션 탐지 (219 패턴)                           │
│      ├─ PII 자동 가명처리 (8종, 형식 보존)                         │
│      └─ DLP 게이트웨이 (10종 민감 데이터)                          │
│                        ↓                                        │
│  [2] AI 생성 ────────────────────────────── GPT-4o              │
│      ├─ 구조 분석 기반 프롬프트 빌드                                │
│      ├─ 가명처리된 텍스트로 API 호출                                │
│      └─ 다중 페이지 배치 생성                                      │
│                        ↓                                        │
│  [3] 출력 보안 ──────────────────────────── AEGIS v5.2          │
│      ├─ 유해 콘텐츠 필터링                                        │
│      ├─ 자격증명/코드 인젝션 차단                                   │
│      └─ PII 세션 기반 자동 복원                                    │
│                        ↓                                        │
│  [4] 할루시네이션 검증 ──────────────────── TruthAnchor v2.0     │
│      ├─ 6 도메인 가드레일 (75 룰)                                 │
│      ├─ 수치 교차검증 (13 패턴, 95 맥락)                           │
│      └─ 문장별 신뢰도 점수 + 교정 제안                              │
```

### 8. 적용 분야

오픈한글AI는 **정확성과 책임성이 중요한 영역**에서 AI 문서 편집의 실사용
가능성을 높이기 위해 설계되었다.

- **공공기관**: 정책 보고서, 민원 응답, 행정 문서 — AI 기본법 준수와 행안부
  가이드라인 대응
- **금융권**: 투자 보고서, 리스크 분석, 고객 안내문 — 금융 도메인 가드레일로
  수치 오류·불완전 공시 방지
- **의료·제약**: 임상 보고서, 환자 안내문 — 의료 가드레일로 투약 오류·부작용
  누락 검증
- **국방·안보**: 작전 보고서, 브리핑 자료 — 기밀 정보 유출 방지 가드레일
- **기업 일반**: 사업 계획서, 제안서, 대외 설명 자료 — 사실 검증과 컴플라이언스
  자동화
